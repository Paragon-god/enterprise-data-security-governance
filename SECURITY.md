# Security Policy

## Supported content

This repository is a portfolio prototype and does not operate a public production service.

Security review focuses on the confidentiality and integrity of published artefacts.

## Report a concern

Report concerns privately to the repository owner through the contact method listed on the GitHub profile.

Do not publish a public issue that contains:

* credentials;
* connection strings;
* personal data;
* temporary application links;
* screenshots containing account details; or
* information that could expose an active environment.

## Public repository rules

The repository must not contain:

* live customer or behavioural data;
* secret values;
* database users or passwords;
* raw environment configuration;
* identity information that is not necessary for the portfolio;
* unsanitised logs or screenshots; or
* production network details.

## Security limitations

The repository demonstrates control design and prototype implementation. It does not provide a certified production security architecture, penetration test, regulatory opinion, or operational assurance statement.
